# 🔒 Security Policy

## Reporting a Vulnerability

**🚨 DO NOT OPEN A PUBLIC ISSUE FOR SECURITY VULNERABILITIES**

## Supported Versions

| Version | Supported | Security Updates |
|:--------|:---------:|:----------------:|
| 2.0.x   | ✅ Yes    | ✅ Active        |
| 1.5.x   | ⚠️ Limited| ⚠️ Critical only |
| 1.0.x   | ❌ No     | ❌ None          |
| < 1.0   | ❌ No     | ❌ None          |

## Security Features in GhostRoute Pro

- ✅ Rate limiting built-in
- ✅ Respects robots.txt
- ✅ Ethical User-Agent headers
- ✅ Configurable timeouts
- ✅ No data storage by default

## Responsible Disclosure Process

1. **Report** - Send details to security@ghostroute.dev
2. **Acknowledge** - We respond within 48 hours
3. **Investigate** - We verify and assess impact
4. **Fix** - Develop and test patch (target: 7 days)
5. **Release** - Publish patched version
6. **Disclose** - Public announcement after 30 days

## Bug Bounty

Currently, we do not offer monetary rewards for GhostRoute itself, but:
- 🏆 All reporters are credited in our Hall of Fame
- 📜 CVEs assigned for qualifying vulnerabilities
- ⭐ Public recognition in release notes
